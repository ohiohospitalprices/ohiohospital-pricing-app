API ENDPOINTS SUMMARY
================================================================================

GET /health
  Purpose: Health check / uptime monitoring
  Response: {status, timestamp}
  Use for: Load balancers, monitoring services

GET /api/search?query=...&hospital=...&limit=50
  Purpose: Search procedures by name and hospital
  Parameters:
    - query (required): procedure name (min 2 chars)
    - hospital (optional): filter by hospital
    - limit (optional): max results (default 50, max 500)
  Response: {query, results[], count}
  Time: 50-100ms

GET /api/compare?procedure=...&hospital=...
  Purpose: Compare price across all hospitals
  Parameters:
    - procedure (required): procedure code or name
    - hospital (optional): highlight this hospital
  Response: {procedure, statistics, all_hospitals[]}
  Time: 30-50ms

GET /api/hospitals
  Purpose: List all hospitals in database
  Response: {hospitals[], count}
  Time: <10ms

GET /api/stats
  Purpose: Database statistics and data freshness
  Response: {statistics: {total_procedures, total_hospitals, ...}}
  Time: <10ms

POST /api/reload
  Purpose: Force reload data from hospital sources
  Response: {status: "success", message}
  Note: Admin endpoint (restrict in production)
  Time: 2-5 seconds

DATABASE STRUCTURE
================================================================================

SQLite Database File: hospital_pricing.db (created on startup)
Location: Configurable via DATABASE_PATH env var

Tables:
  1. procedures
     - id (PK)
     - procedure_code (indexed, unique)
     - procedure_name (indexed)
     - description
     - created_at

  2. hospitals
     - id (PK)
     - name (unique)
     - url
     - last_updated

  3. hospital_pricing
     - id (PK)
     - hospital_id (FK, indexed)
     - procedure_id (FK, indexed)
     - price
     - currency
     - updated_at
     - Unique constraint: (hospital_id, procedure_id)

Indexes:
  - idx_procedure_name (fast search)
  - idx_procedure_code (code lookup)
  - idx_hospital_id (filtering)
  - idx_hospital_procedure (unique constraint)

Current Data:
  - Hospitals: 15
  - Procedures: ~3,200
  - Pricing Records: ~48,000
